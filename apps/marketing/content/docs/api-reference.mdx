## Base URL

```
http://localhost:8080/v1
```

All endpoints require authentication via session cookie (set after GitHub OAuth flow).

## Authentication

### GitHub OAuth

```
GET /v1/auth/github/login
```
Redirects to GitHub OAuth. After authorization, GitHub redirects to the callback.

```
GET /v1/auth/github/callback?code={code}&state={state}
```
Exchanges code for access token, creates session, redirects to dashboard.

## Repositories

### List repositories

```
GET /v1/github/repositories
```

Returns all repositories accessible via the GitHub App installation.

### Connect repository

```
POST /v1/github/repositories/{repoID}/connect
```

Connects a repository to Arrakis and queues an initial dependency sync.

### Disconnect repository

```
DELETE /v1/github/repositories/{repoID}
```

### Assign policy

```
PUT /v1/github/repositories/{repoID}/policy
Body: { "policy_id": 42 }
```

### Trigger dependency sync

```
POST /v1/github/repositories/{repoID}/deps/sync
Body: { "force": false }
```

### Trigger scan

```
POST /v1/github/repositories/{repoID}/scans/run
```

## Policies

### List policies

```
GET /v1/policies
```

### Get policy

```
GET /v1/policies/{policyID}
```

### Create policy

```
POST /v1/policies
```

```json
{
  "name": "Production",
  "enabled": true,
  "triggers": [
    { "type": "schedule", "cron": "0 2 * * *", "timezone": "UTC" }
  ],
  "sources": {
    "osv_enabled": true,
    "ghsa_enabled": true,
    "nvd_enabled": false,
    "supply_chain_enabled": true
  }
}
```

### Update policy

```
PUT /v1/policies/{policyID}
```

### Delete policy

```
DELETE /v1/policies/{policyID}
```

### Trigger policy scan

```
POST /v1/policies/{policyID}/scans/run
```

Queues a scan for all repositories assigned to this policy.

## Scans

### List scans

```
GET /v1/scans
```

Query parameters: `repo_id`, `status`, `limit`, `page`

### Get scan

```
GET /v1/scans/{scanID}
```

Returns scan run metadata and all findings.

### Stream AI logs

```
GET /v1/scans/{scanID}/ai/logs/stream
```

Server-Sent Events stream of AI analyzer output for the scan.

## Findings

### List findings

```
GET /v1/findings
```

Query parameters:

| Parameter | Values |
|---|---|
| `severity` | `critical`, `high`, `medium`, `low` |
| `status` | `open`, `resolved` |
| `manager` | `npm`, `pip`, `go` |
| `repo_id` | Repository ID |
| `limit` | Default: 50 |
| `page` | Pagination offset |

### Get finding

```
GET /v1/findings/{findingID}
```

### Update finding status

```
PATCH /v1/findings/{findingID}
Body: { "status": "resolved" }
```

## Integrations

### List integrations

```
GET /v1/integrations
```

### Create integration

```
POST /v1/integrations
Body: { "provider": "slack", "name": "Security alerts", "config": {...} }
```

### Delete integration

```
DELETE /v1/integrations/{integrationID}
```

### Create issue from finding

```
POST /v1/integrations/{provider}/issues
Body: { "finding_id": 12345 }
```

`{provider}`: `github`, `jira`, `linear`

### Get integration activity

```
GET /v1/integrations/{integrationID}/activity
```

## Custom Domains

### List domains

```
GET /v1/domains
```

### Add domain

```
POST /v1/domains
Body: { "hostname": "security.example.com" }
```

Returns DNS records to create for verification.

### Verify domain

```
POST /v1/domains/{domainID}/verify
```

### Delete domain

```
DELETE /v1/domains/{domainID}
```
