## Why a GitHub App?

Arrakis uses a GitHub App (in addition to OAuth) to:

1. **Access repository contents** — Read manifest files without requiring a user's personal token
2. **Operate at installation level** — Works across organizations, not just personal accounts
3. **Use short-lived tokens** — Installation access tokens expire in 1 hour; no long-lived credentials stored

The GitHub App requires **read-only** access to repository contents and metadata.

## Creating your GitHub App

When self-hosting, you need to create your own GitHub App and configure its credentials in Arrakis.

1. Go to [GitHub → Settings → Developer settings → GitHub Apps](https://github.com/settings/apps/new)
2. Configure:
   - **App name**: `Arrakis Security` (or any name)
   - **Homepage URL**: Your Arrakis instance URL
   - **Callback URL**: `https://yourdomain.com/v1/auth/github/app/callback`
   - **Webhook URL**: Leave blank (not required)
   - **Permissions**:
     - Repository contents: Read-only
     - Repository metadata: Read-only

3. Generate and download a private key
4. Note the App ID from the app settings page

## Configuring Arrakis

Set these environment variables:

```bash
GITHUB_APP_ID=123456
GITHUB_APP_PRIVATE_KEY_PATH=/path/to/private-key.pem
# Or inline base64:
GITHUB_APP_PRIVATE_KEY_BASE64=<base64-encoded-pem>
```

## How token creation works

When Arrakis needs to access a repository, it:

1. **Creates a signed JWT** using the App's RSA private key
   - Issued by: `GITHUB_APP_ID`
   - Expires: 10 minutes
   
2. **Exchanges the JWT** for an installation access token
   - `POST https://api.github.com/app/installations/{installationID}/access_tokens`
   - Token expires in 1 hour
   
3. **Uses the installation token** for all repository API calls
   - `GET /repos/{owner}/{repo}/git/trees/{sha}?recursive=1`
   - `GET /repos/{owner}/{repo}/contents/{path}`

Tokens are not cached — a new installation token is created for each dependency sync.

## Installation flow (user side)

When a user connects their account:

1. They click **Install GitHub App** in Arrakis Settings
2. They're redirected to GitHub's app installation page
3. They choose personal account or organization, and which repos to grant access to
4. GitHub redirects back to `GET /v1/auth/github/app/install`
5. Arrakis stores the `UserGithubInstallation` record with the `installation_id`

## Repository access

After installation, Arrakis can access any repository the user granted during step 3. To grant access to additional repositories, the user reinstalls or updates the installation on GitHub.

```
GET /v1/github/repositories
```

Returns all repositories accessible via the user's GitHub App installation. Only repositories that have been installed will appear.

## Troubleshooting

**"Installation not found"** — The user has not installed the GitHub App, or has revoked access. Direct them to Settings → GitHub App → Install.

**"401 Unauthorized" when syncing** — The private key may be misconfigured. Verify `GITHUB_APP_ID` matches the key.

**"Repository not accessible"** — The repository may not be included in the installation scope. The user should update their GitHub App installation to include it.
