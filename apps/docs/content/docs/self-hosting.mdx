## Development setup

Clone the repo and start the dev stack:

```bash
git clone git@github.com:chann44/TGE.git
cd arrakis

cp .env.example .env
docker compose -f deployments/dev.compose.yml up -d
```

The API is available at `http://localhost:8080`.

## Production setup

The production compose uses Traefik for TLS termination and serves the web UI and API on a single domain.

```bash
git clone git@github.com:chann44/TGE.git
cd arrakis

cp .env.example .env
```

Set the required environment variables in `.env`:

```bash
# Traefik / TLS
TRAEFIK_ACME_EMAIL=ops@example.com
DOMAIN_RECORD_TYPE=A
DOMAIN_RECORD_VALUE=<your-server-public-ip>

# GitHub OAuth App
GITHUB_CLIENT_ID=
GITHUB_CLIENT_SECRET=
GITHUB_REDIRECT_URI=https://yourdomain.com/v1/auth/github/callback

# GitHub App
GITHUB_APP_ID=
GITHUB_APP_PRIVATE_KEY_PATH=

# Database
DATABASE_URL=postgres://arrakis:arrakis@postgres:5432/arrakis

# Redis
REDIS_URL=redis://redis:6379

# Optional: AI Analyzer
OPENAI_API_KEY=
AI_ANALYZER_ENABLED=true
```

Then start:

```bash
docker compose -f deployments/selfhost.compose.yml up -d
```

## Domain configuration

After startup, open `http://<server-ip>/settings` and add your domain. Arrakis will display the DNS records you need to create:

| Record | Type | Value |
|---|---|---|
| `_tge-challenge.<domain>` | TXT | Ownership token |
| `<domain>` | A | Your server IP |

After DNS propagates, click **Verify**. Traefik will automatically provision a Let's Encrypt certificate.

<Callout type="warn">
  Ensure ports 80 and 443 are open on your host firewall before starting the stack. Traefik needs both for the ACME challenge.
</Callout>

## Docker images

The default compose uses published images:

```
chann44/tge-backend:latest
chann44/tge-web:latest
```

To build and publish your own:

```bash
docker login
make docker-build
make docker-push
```

To override the image names:

```bash
export TGE_BACKEND_IMAGE=your-registry/tge-backend:latest
export TGE_WEB_IMAGE=your-registry/tge-web:latest
docker compose -f deployments/selfhost.compose.yml up -d
```

## Log streaming (optional)

Arrakis supports streaming system health logs to ClickHouse:

```bash
CLICKHOUSE_HOST=localhost
CLICKHOUSE_PORT=9000
CLICKHOUSE_USER=default
CLICKHOUSE_PASSWORD=clickhouse
CLICKHOUSE_DATABASE=default
```

Without ClickHouse, logs are written to stdout only.

## Infrastructure requirements

| Component | Minimum | Recommended |
|---|---|---|
| CPU | 2 vCPU | 4 vCPU |
| RAM | 2 GB | 8 GB |
| Disk | 20 GB | 100 GB |
| Ports | 80, 443 | 80, 443 |

The AI Analyzer container requires an additional 512 MB RAM per concurrent scan if enabled.
