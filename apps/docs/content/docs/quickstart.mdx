## Prerequisites

- A running Arrakis instance (see [Self-hosting](/docs/self-hosting))
- A GitHub account with at least one repository
- The Arrakis GitHub App installed on your account or organization

## 1. Authenticate with GitHub

Navigate to your Arrakis instance and click **Sign in with GitHub**. Arrakis uses GitHub OAuth — it requests read access to your repositories and email.

```text
GET /v1/auth/github/login
```

After authorizing, you'll be redirected back to the dashboard.

## 2. Install the GitHub App

Arrakis uses a GitHub App to access repository contents for dependency extraction. This is separate from OAuth.

Go to **Settings → GitHub App** and click **Install App**. Choose whether to install on your personal account or an organization, and select which repositories to grant access to.

<Callout type="info">
  Arrakis only requests read access to repository contents and metadata. It never writes to your repositories.
</Callout>

## 3. Connect a repository

From the **Repositories** page, you'll see all repos accessible via your GitHub App installation. Click **Connect** next to any repository.

When you connect a repo, Arrakis immediately queues a **dependency sync** — it clones the repo tree, finds manifest files, and builds the dependency graph. This typically takes 10–60 seconds.

```text
POST /v1/github/repositories/{repoID}/connect
```

## 4. Run your first scan

Once the dependency sync completes, click **Run Scan** on the repository page. Arrakis will:

1. Load the dependency graph (direct + transitive to depth 4)
2. Query OSV, GHSA, and NVD for known vulnerabilities
3. Run supply chain analysis on all direct dependencies
4. Persist and deduplicate findings

```text
POST /v1/github/repositories/{repoID}/scans/run
```

Scan time depends on the size of your dependency graph. A typical Node.js project with 500 transitive deps takes 30–90 seconds.

## 5. Review findings

Navigate to the scan results to see findings grouped by severity:

| Severity | Threshold |
|---|---|
| **Critical** | Risk score ≥ 50 or critical CVE |
| **High** | Risk score ≥ 30 or high CVE |
| **Medium** | Risk score ≥ 15 or medium CVE |
| **Low** | CVSS < 4.0 |

Each finding includes:
- Package name, manager, and resolved version
- Advisory ID (CVE/GHSA/GHSA alias)
- Title and summary
- Fixed version (if available)
- Sources that detected it (OSV, GHSA, NVD, supply chain)

## 6. Configure a policy (optional)

By default, repos use a global default policy. To customise which advisory sources are queried, set up automated scans, or configure SAST settings, create a **Policy** and assign it to your repo.

See [Policies](/docs/policies) for details.
