## What is a policy?

A **Policy** is a named, reusable configuration that controls how Arrakis scans a repository. It defines:

- **When** to scan (triggers: schedule, manual, push)
- **What** to query (advisory sources: OSV, GHSA, NVD, custom feeds)
- **How** to analyze (SAST settings, AI analyzer config)
- **Where** to push findings (registry and integration settings)

Policies are created once and assigned to one or more repositories. Repositories not explicitly assigned a policy use the default global policy.

## Creating a policy

```text
POST /v1/policies
```

```json
{
  "name": "Production — Strict",
  "enabled": true,
  "triggers": [
    {
      "type": "schedule",
      "cron": "0 2 * * *",
      "timezone": "America/New_York"
    }
  ],
  "sources": {
    "osv_enabled": true,
    "ghsa_enabled": true,
    "nvd_enabled": true,
    "govulncheck_enabled": true,
    "supply_chain_enabled": true
  },
  "sast": {
    "enabled": false,
    "ai_enabled": true,
    "ai_max_files_per_scan": 50
  }
}
```

## Policy structure

### Triggers

Controls when automated scans run. A policy can have multiple triggers.

```json
"triggers": [
  {
    "type": "schedule",
    "cron": "0 2 * * *",
    "timezone": "UTC"
  }
]
```

See [Triggers](/docs/triggers) for all supported types and configuration.

### Sources

Controls which advisory databases are queried during a scan.

```json
"sources": {
  "osv_enabled": true,
  "ghsa_enabled": true,
  "nvd_enabled": false,
  "govulncheck_enabled": true,
  "supply_chain_enabled": true,
  "registry_first": false,
  "registry_max_age_days": 7
}
```

| Field | Default | Description |
|---|---|---|
| `osv_enabled` | `true` | Query Open Source Vulnerabilities database |
| `ghsa_enabled` | `true` | Query GitHub Security Advisories |
| `nvd_enabled` | `false` | Query National Vulnerability Database (requires API key) |
| `govulncheck_enabled` | `false` | Run govulncheck for Go projects |
| `supply_chain_enabled` | `true` | Run behavioral supply chain analysis |
| `registry_first` | `false` | Check registry metadata before advisory lookups |
| `registry_max_age_days` | `7` | Days before registry metadata is considered stale |

See [Advisory Sources](/docs/advisory-sources) for details on each source.

### Custom sources

Add your own advisory feeds alongside the built-in sources:

```json
"custom_sources": [
  {
    "name": "Internal Security Feed",
    "url": "https://security.internal/advisories/osv.json",
    "format": "osv",
    "auth_header": "Bearer ${INTERNAL_TOKEN}"
  }
]
```

Supported formats: `osv`, `nvd`.

### SAST

Static analysis and AI analyzer configuration:

```json
"sast": {
  "enabled": false,
  "patterns_enabled": false,
  "rulesets": [],
  "min_severity": "medium",
  "ai_enabled": true,
  "ai_max_files_per_scan": 100,
  "ai_reachability": false,
  "ai_suggest_fix": false
}
```

### Registry

Controls interaction with private package registries:

```json
"registry": {
  "push_enabled": false,
  "pull_enabled": false,
  "pull_trusted_keys": []
}
```

## Assigning a policy to a repository

```text
PUT /v1/github/repositories/{repoID}/policy
Body: { "policy_id": 42 }
```

A repository can only have one active policy at a time. Assigning a new policy replaces the previous assignment.

## Running a policy scan manually

```text
POST /v1/policies/{policyID}/scans/run
```

This queues a scan for all repositories assigned to the policy with trigger type `manual`.

## Updating a policy

```text
PUT /v1/policies/{policyID}
```

Policy updates take effect on the next scan — in-progress scans are not affected.

## Deleting a policy

```text
DELETE /v1/policies/{policyID}
```

Repositories assigned to a deleted policy revert to the default global policy.
