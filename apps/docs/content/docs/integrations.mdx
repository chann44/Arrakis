## Overview

Arrakis can push findings to external tools after a scan completes. Integrations are configured per user and can be enabled or disabled individually.

| Integration | Type | Use case |
|---|---|---|
| **Slack** | Webhook | Scan completion notifications |
| **Discord** | Webhook | Scan completion notifications |
| **GitHub Issues** | API | Create issues from findings |
| **Jira** | API | Create tickets from findings |
| **Linear** | GraphQL | Create issues from findings |

## Webhook integrations (Slack, Discord)

Webhooks receive a notification when a scan completes. The payload includes the scan summary and finding counts.

### Slack

1. Create an [Incoming Webhook](https://api.slack.com/messaging/webhooks) in your Slack workspace
2. In Arrakis Settings → Integrations, add a new Slack integration with the webhook URL
3. Configure the channel and message format

Arrakis sends a structured message with:
- Repository name and scan trigger
- Finding counts by severity
- Link to the scan results

### Discord

Same setup as Slack — create a Discord channel webhook and add it in Arrakis Settings.

## Issue tracker integrations

Issue trackers (GitHub, Jira, Linear) are triggered manually from the finding detail view, or automatically based on policy configuration.

### GitHub Issues

Arrakis creates issues in any GitHub repository you have write access to (via your OAuth token).

**Setup:**
1. Settings → Integrations → GitHub
2. Arrakis will use your existing OAuth token — no additional credentials needed
3. Optionally configure default labels and assignees

**Issue format:**
```text
[SECURITY] lodash@4.17.20 — Command Injection (CVE-2021-23337)

Severity: High
Package: lodash 4.17.20 (npm)
Fixed in: 4.17.21

## Summary
lodash prior to 4.17.21 is vulnerable to Command Injection...

## References
- https://github.com/advisories/GHSA-35jh-r3h4-6jhm
- https://nvd.nist.gov/vuln/detail/CVE-2021-23337

---
Detected by Arrakis | scan #789
```

### Jira

**Setup:**
1. Settings → Integrations → Jira
2. Enter your Jira base URL, project key, and API token
3. Jira issues are created via the [Jira REST API v3](https://developer.atlassian.com/cloud/jira/platform/rest/v3/)

Arrakis maps finding severity to Jira priority:

| Finding severity | Jira priority |
|---|---|
| Critical | Highest |
| High | High |
| Medium | Medium |
| Low | Low |

### Linear

**Setup:**
1. Settings → Integrations → Linear
2. Enter your Linear API key and select the default team
3. Issues are created via the [Linear GraphQL API](https://developers.linear.app/docs/graphql/working-with-the-graphql-api)

Arrakis creates Linear issues with the appropriate priority label and links to the finding in Arrakis.

## Integration activity log

All integration actions are logged in `IntegrationActivity`:

```text
GET /v1/integrations/{integrationID}/activity
```

Each activity record includes:
- Action type (e.g., `create_issue`, `send_notification`)
- Status (`success`, `failed`)
- Detail message
- Timestamp

Use the activity log to debug failed integration calls.

## Creating an issue from a finding

```text
POST /v1/integrations/{provider}/issues
Content-Type: application/json

{
  "finding_id": 12345
}
```

Supported providers: `github`, `jira`, `linear`

The response includes the URL of the created issue.
